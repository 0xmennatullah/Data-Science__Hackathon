## рџЏ† Hackathon вЂ” Vehicle Sales (~558k auction records, Jan 2014 вЂ“ Jul 2015)

### V1 вЂ” Quick exploration

Two separate notebooks for cleaning and insights.

```bash
cd hackathon/V1
jupyter notebook data_cleaning.ipynb
jupyter notebook insights.ipynb
```

### V2 вЂ” Full pipeline (recommended)

A single end-to-end notebook + a live interactive dashboard.

**Notebook** (cleaning в†’ EDA в†’ inline charts):

```bash
cd hackathon/V2/insights
jupyter notebook vehicle-analysis.ipynb
```

Run cells top to bottom with **Shift + Enter**, or use **Cell в†’ Run All**.

What it covers:
- Raw data audit & missing value visualisation
- Cleaning вЂ” fix mislabels, parse dates, IQR outlier removal, standardise casing
- Feature engineering вЂ” `VehicleAge`, `PriceDiff`, `PriceBand`, etc.
- EDA вЂ” market overview, pricing intelligence, time trends, correlations

**Dashboard** (Plotly Dash, fully interactive):

```bash
cd hackathon/V2/insights
python dashboard.py
```

Then open в†’ **http://127.0.0.1:8050**

Sidebar filters (Make, Body, State, Transmission, Year range) update all 10 charts live:
- KPI cards, price distribution, top makes, body type share
- MMR vs Selling Price scatter (colored by % above/below market)
- Condition score vs price, monthly volume + revenue time series
- Price by model year, above/below MMR by make, Make Г— Body heatmap

Stop with **Ctrl + C**.

## рџ—‚пёЏ Key Column Reference (Vehicle Sales)

| Column | Description |
|--------|-------------|
| `SellingPrice` | Final auction sale price ($) |
| `MMR` | Manheim Market Report вЂ” estimated market value |
| `ConditionValue` | Vehicle condition score (1вЂ“50 scale) |
| `Odometer` | Mileage at time of sale |
| `Year` | Model year of the vehicle |
| `SaleDate` | Date of auction sale |
| `PriceDiff` | `SellingPrice - MMR` (engineered) |
| `PriceDiffPct` | `PriceDiff / MMR * 100` (engineered) |
| `VehicleAge` | `SaleYear - Year` (engineered) |
